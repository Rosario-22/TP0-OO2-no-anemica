Concepto del artículo de *Martin Fowler*.
Un objeto no anémico contiene atributos y comportamiento. 
La lógica de negocio relacionada con sus datos se encuentra dentro del mismo objeto, respetando el encapsulamiento.
A diferencia del modelo anémico donde se delega la lógica a clases externas, este enfque representa la verdadera OO
ya que combina estado y comportamiento dentro de un  mismo objeto.
